# Container Images

Anyclaw publishes two container images to GitHub Container Registry (ghcr.io) on every semver tag.

## Images

| Image | Contents | Use Case |
|-------|----------|----------|
| `ghcr.io/donbader/anyclaw` | anyclaw binary only (distroless) | Minimal deployments where you supply your own extensions |
| `ghcr.io/donbader/anyclaw-ext` | All built-in ext/ binaries (distroless) | Ready-to-use extensions — `COPY --from=` into your image |

## Tags

Every release produces these tags per image:

| Tag | Example | Description |
|-----|---------|-------------|
| semver | `0.2.0` | Exact release version |
| major.minor | `0.2` | Tracks latest patch in a minor series |
| short-SHA | `a1b2c3d` | Git commit SHA for traceability |
| `latest` | `latest` | Most recent release from default branch |

Tags are generated by [docker/metadata-action](https://github.com/docker/metadata-action) in the Docker CI workflow.

## Platforms

All images are multi-arch manifests supporting:

- `linux/amd64`
- `linux/arm64`

Verified on every release via `docker buildx imagetools inspect`.

## Security Scanning

[Trivy](https://github.com/aquasecurity/trivy) runs on every image push. Findings at `CRITICAL` or `HIGH` severity block the release.

## Retention Policy

A weekly cleanup workflow (`.github/workflows/ghcr-cleanup.yml`) manages image lifecycle:

- Untagged manifests (build intermediates) are deleted
- Last 10 versions are always kept
- `latest` and semver tags are preserved
- Runs every Sunday at 03:00 UTC, also available via manual dispatch

## Usage

Use the ext image to compose your own setup:

```dockerfile
FROM ghcr.io/donbader/anyclaw:latest AS core
FROM ghcr.io/donbader/anyclaw-ext:latest AS ext

FROM node:20-slim AS my-app
COPY --from=core /usr/local/bin/anyclaw /usr/local/bin/anyclaw
COPY --from=ext /usr/local/bin/channels/telegram /usr/local/bin/channels/telegram
COPY --from=ext /usr/local/bin/tools/system-info /usr/local/bin/tools/system-info
```

Binary layout in the ext image follows the `@built-in/` convention:

```
/usr/local/bin/
├── agents/
│   └── mock-agent
├── channels/
│   ├── telegram
│   └── debug-http
└── tools/
    └── system-info
```
