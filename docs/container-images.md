# Container Images

Protoclaw publishes two container images to GitHub Container Registry (ghcr.io) on every semver tag.

## Images

| Image | Contents | Use Case |
|-------|----------|----------|
| `ghcr.io/donbader/protoclaw` | protoclaw binary only (distroless) | Minimal deployments where you supply your own extensions |
| `ghcr.io/donbader/protoclaw-builder` | protoclaw + all ext/ binaries | Ready-to-run with all built-in agents, channels, and tools |

## Tags

Every release produces these tags per image:

| Tag | Example | Description |
|-----|---------|-------------|
| semver | `0.2.0` | Exact release version |
| major.minor | `0.2` | Tracks latest patch in a minor series |
| short-SHA | `a1b2c3d` | Git commit SHA for traceability |
| `latest` | `latest` | Most recent release from default branch |

Tags are generated by [docker/metadata-action](https://github.com/docker/metadata-action) in the Docker CI workflow.

## Platforms

All images are multi-arch manifests supporting:

- `linux/amd64`
- `linux/arm64`

Verified on every release via `docker buildx imagetools inspect`.

## Security Scanning

[Trivy](https://github.com/aquasecurity/trivy) runs on every image push. Findings at `CRITICAL` or `HIGH` severity block the release.

## Retention Policy

A weekly cleanup workflow (`.github/workflows/ghcr-cleanup.yml`) manages image lifecycle:

- Untagged manifests (build intermediates) are deleted
- Last 10 versions are always kept
- `latest` and semver tags are preserved
- Runs every Sunday at 03:00 UTC, also available via manual dispatch

## Usage

Pull the builder image (includes all extensions):

```sh
docker pull ghcr.io/donbader/protoclaw-builder:latest
```

Use as a base for custom images:

```dockerfile
FROM ghcr.io/donbader/protoclaw-builder:latest AS builder

FROM node:20-slim AS my-app
COPY --from=builder /usr/local/bin/protoclaw /usr/local/bin/protoclaw
COPY --from=builder /usr/local/bin/channels/telegram /usr/local/bin/channels/telegram
COPY --from=builder /usr/local/bin/tools/system-info /usr/local/bin/tools/system-info
```

Binary layout in the builder image follows the `@built-in/` convention:

```
/usr/local/bin/
├── protoclaw
├── agents/
│   ├── mock-agent
│   └── opencode-wrapper
├── channels/
│   ├── telegram
│   └── debug-http
└── tools/
    └── system-info
```
