## v7.0 Requirements

**Defined:** 2026-04-11
**Goal:** Tech debt reduction, dependency modernization, architecture deduplication, and CI hardening.

### DEPS — Dependency Modernization

- [ ] **DEPS-01**: Remove `async-trait` dependency from all 9 workspace crates, replace with native async fn in traits (Rust 1.85 MSRV supports this). SDK crate trait changes require semver-compatible version bump to 0.2.0.
- [ ] **DEPS-02**: Apply pending patch updates: rustls-webpki, wasm-bindgen ecosystem, rand, cc, js-sys/web-sys.
- [ ] **DEPS-03**: Trim `tokio` features in library crates — replace `features = ["full"]` with minimal required features (e.g., `["sync"]`, `["rt", "macros"]`).

### ERR — Error Handling

- [ ] **ERR-01**: Replace `anyhow::Result` in `protoclaw-supervisor` public API (`run()`, `run_with_cancel()`, `boot_managers()`) with typed `SupervisorError` (thiserror enum).
- [ ] **ERR-02**: Remove `unsafe` blocks in `crates/protoclaw/src/init.rs` test code (lines 93, 97) — replace `std::env::set_var` with `temp-env` crate or serial test isolation.

### ARCH — Architecture Deduplication

- [ ] **ARCH-01**: Extract shared slot pattern (`cancel_token`, `backoff`, `crash_tracker`, `disabled`) from AgentSlot, ChannelSlot, and ManagerSlot into a generic `SubprocessSlot<C>` in `protoclaw-core`.
- [ ] **ARCH-02**: Unify crash recovery state machine (record crash → check loop → backoff → respawn) into a shared `CrashRecovery` helper used by agents, channels, and supervisor managers.
- [ ] **ARCH-03**: Fix visibility — `PendingPermission` to `pub(crate)`, `AgentSlot` fields to `pub(crate)`, remove `with_log_level()` duplication.

### CFG — Config Cleanup

- [ ] **CFG-01**: Replace stringly-typed config fields with enums: `ToolConfig.tool_type` → `ToolType::Mcp | ToolType::Wasm`, `log_format` → `LogFormat::Pretty | LogFormat::Json`, `reaction_lifecycle` → `ReactionLifecycle::Remove | ReactionLifecycle::ReplaceDone`.
- [ ] **CFG-02**: Clarify crash config hierarchy — document or rename `SupervisorConfig.max_restarts`/`restart_window_secs` vs per-entity `CrashTrackerConfig` to prevent confusion.

### QUAL — Code Quality

- [ ] **QUAL-01**: Extract `collect_channel_message` (~183 lines) in `protoclaw-channels/src/manager.rs` into named sub-functions.
- [ ] **QUAL-02**: Extract `spawn_inner` (~159 lines) and `spawn` (~168 lines) in agents connection/docker_backend into smaller functions.
- [ ] **QUAL-03**: Reduce `SessionKey` cloning in hot message-routing paths — evaluate `Arc<str>` or `Copy` newtype.
- [ ] **QUAL-04**: Extract remaining functions >100 lines: `handle_incoming`, `handle_crash`, `handle_channel_event`, `start` (tools manager), `validate_config`, custom `deserialize<D>`.

### CI — CI Hardening

- [ ] **CI-01**: Add `cargo audit` step to CI workflow for security advisory checking.
- [ ] **CI-02**: Add MSRV enforcement job — build with `rust-version = "1.85"` toolchain to catch accidental use of newer features.
- [ ] **CI-03**: Add `--locked` flag to all `cargo build`/`cargo test` invocations in CI to prevent Cargo.lock drift.

### TEST — Test Coverage

- [ ] **TEST-01**: Add unit tests for `protoclaw-sdk-agent` — test AgentAdapter trait hooks (7 default methods), GenericAcpAdapter passthrough behavior.
- [ ] **TEST-02**: Add unit tests for `protoclaw-sdk-tool` — test Tool trait contract, ToolServer dispatch routing, error handling paths.
- [ ] **TEST-03**: Add `#[cfg(test)]` modules to files with logic but no tests: `backend.rs`, `agents_command.rs`, `tools_command.rs`, `generic.rs`.

### DOCS — Documentation Lints

- [ ] **DOCS-01**: Add `#![warn(missing_docs)]` to all 4 SDK crate `lib.rs` files and fix resulting warnings with doc comments on all public items.
- [ ] **DOCS-02**: Add doc comments to `protoclaw-sdk-types` public types (ChannelEvent, SessionKey, PermissionOption, ChannelCapabilities, ContentKind, ThoughtContent).

## v7.1 Requirements

**Defined:** 2026-04-11
**Goal:** Shrink protoclaw Docker images to competitive sizes, restructure extension architecture, optimize agent images, and establish ghcr.io lifecycle management.

### IMG — Core Image

- [ ] **IMG-01**: Core Docker image contains only the `protoclaw` binary + distroless base (no extension binaries)
- [ ] **IMG-02**: `[profile.release]` includes `strip = true` and `lto = true` in workspace Cargo.toml
- [ ] **IMG-03**: Dockerfile includes binary size validation gate (reject suspiciously small binaries)
- [ ] **IMG-04**: Core image size under 30MB (distroless ~3MB + stripped/LTO protoclaw binary)

### EXT — Extension Architecture

- [ ] **EXT-01**: `@built-in/` path resolution follows `extensions_dir/{agents,channels,tools}/<name>` convention consistently (e.g. `@built-in/agents/opencode-wrapper`, `@built-in/channels/telegram`)
- [ ] **EXT-02**: Builder image published to ghcr.io containing all compiled ext/ binaries — users `COPY --from=` to compose their own images
- [ ] **EXT-03**: Example Dockerfiles demonstrate composing custom images from core + builder (COPY channels, agents as needed)
- [ ] **EXT-04**: Root Dockerfile restructured — core stage (protoclaw only), builder stage (all ext/ binaries for COPY --from=)

### OPT — Extension Image Optimization

- [ ] **OPT-01**: Example opencode agent image optimized — multi-stage npm install with production-only deps, node pruning
- [ ] **OPT-02**: Example claude-code agent image optimized with same approach
- [ ] **OPT-03**: opencode-agent image target size reduction documented (baseline 802MB → target)

### GHCR — Registry Management

- [ ] **GHCR-01**: Tagging conventions documented and enforced — semver, latest, short-SHA on every push
- [ ] **GHCR-02**: Image retention policy configured — delete untagged manifests after N days, keep last N semver tags
- [ ] **GHCR-03**: Automated cleanup workflow (GitHub Actions) for stale/untagged images
- [ ] **GHCR-04**: Container vulnerability scanning integrated (trivy or similar) on image push
- [ ] **GHCR-05**: Multi-arch builds (amd64 + arm64) verified end-to-end for core + builder images
