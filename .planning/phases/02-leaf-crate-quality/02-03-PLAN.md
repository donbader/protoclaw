**1. Type agents_command.rs** (per D-01, D-02):

The single Value usage is `AgentsCommand::ListSessions.reply`:
```rust
ListSessions {
    agent_name: String,
    reply: oneshot::Sender<Result<serde_json::Value, String>>,
},
```

Replace with the typed `SessionListResult` from `anyclaw-sdk-types::acp`:
```rust
ListSessions {
    agent_name: String,
    reply: oneshot::Sender<Result<anyclaw_sdk_types::SessionListResult, String>>,
},
```

Add `use anyclaw_sdk_types::SessionListResult;` at the top. This is a breaking change for the consumer (anyclaw-agents manager) — acceptable per AGENTS.md ("Internal crates can have breaking changes freely"). The agents manager will be updated in Phase 3.

**2. Error enum audit** (per D-09, D-10):

Review `SupervisorError` in error.rs:
- `BootFailed { manager, reason }` — good, has context
- `ManagerCrashed { manager, reason }` — good
- `ShutdownTimeout { manager }` — good
- `CrashLoop { manager, count, window_secs }` — good, rich context
- `Config(String)` — could be improved to wrap the source error. Change to:
  ```rust
  #[error("configuration error: {0}")]
  Config(#[from] anyclaw_config::ConfigError),
  ```
  BUT — check if `anyclaw-core` depends on `anyclaw-config`. Looking at the crate dependency flow in AGENTS.md: `anyclaw-core` does NOT depend on `anyclaw-config`. So keep `Config(String)` — the string is the right abstraction at this layer. The supervisor (which depends on both) converts ConfigError to string when constructing SupervisorError::Config.

Review `ManagerError` in error.rs:
- `NotStarted` — good
- `AlreadyRunning` — good
- `HealthCheckFailed { reason }` — good
- `SendFailed(String)` — good, wraps mpsc send error
- `Internal(String)` — catch-all. Consider whether any specific variants are needed. Check if callers use `Internal` for distinct failure modes. If so, add typed variants. If it's truly a catch-all for unexpected errors, keep it.

Review `SessionStoreError` in session_store.rs:
- `Backend(String)` — wraps rusqlite errors as strings. Consider `Backend(#[source] Box<dyn std::error::Error + Send + Sync>)` for better error chaining. BUT — the current pattern of `.map_err(|e| SessionStoreError::Backend(e.to_string()))` is used everywhere in sqlite_session_store.rs. Changing to boxed source would require updating all those call sites. Keep `Backend(String)` for now — it's consistent and the string contains the full error message. The D-10 decision says "manual From impls for conversions that need to add context" — the current pattern adds context via the variant name.
- `Serialization(String)` — good, not currently used but reserved for future JSON session data.
