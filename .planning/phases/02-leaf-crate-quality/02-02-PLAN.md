<objective>
Replace all serde_json::Value usages in anyclaw-jsonrpc with typed JSON-RPC structs and upgrade the codec to decode/encode typed messages.

Purpose: The jsonrpc crate is the framing layer — every ACP and channel message passes through it. Typing the codec means upstream crates receive typed JsonRpcMessage instead of raw Value, eliminating manual JSON parsing at every call site.
Output: Typed JSON-RPC types, typed codec, complete error enum, zero Value usages.
</objective>

Replace serde_json::Value in types.rs with typed alternatives (per D-02):

**1. Create `RequestId` enum** to replace `Option<serde_json::Value>` for id fields:
```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(untagged)]
pub enum RequestId {
    Number(i64),
    String(String),
}
```
JSON-RPC 2.0 spec says id MUST be String, Number, or Null. This covers all valid cases without Value.

**2. Update JsonRpcRequest:**
- `id: Option<serde_json::Value>` → `id: Option<RequestId>`
- `params: Option<serde_json::Value>` — this MUST stay as `Option<serde_json::Value>` because params can be any JSON object or array. Per D-03, this is the extensible boundary where typed core meets arbitrary method-specific payloads. Add field-level `#[allow(clippy::disallowed_types)]` with comment: `// Extensible: params schema varies per JSON-RPC method (D-03)`.
- Update `JsonRpcRequest::new()` signature: `id: Option<RequestId>` instead of `Option<serde_json::Value>`.

**3. Update JsonRpcResponse:**
- `id: Option<serde_json::Value>` → `id: Option<RequestId>`
- `result: Option<serde_json::Value>` — MUST stay as Value per D-03 (result schema varies per method). Add field-level allow with comment.
- Update `JsonRpcResponse::success()` and `JsonRpcResponse::error()` signatures to take `Option<RequestId>`.

**4. Update JsonRpcError:**
- `data: Option<serde_json::Value>` — MUST stay as Value per D-03 (error data is implementation-defined). Add field-level allow with comment.

**5. Unwrap cleanup**: The types.rs file has ~10 `.unwrap()` calls — all in test code. Production code has zero unwraps. Verify this is still true after changes.

**6. Error enum audit** (per D-09): Review FramingError in error.rs. Current variants: InvalidHeader, FrameTooLarge, InvalidJson, InvalidUtf8, Io. This is already well-structured with thiserror. Verify no missing variants needed for the typed codec (Task 2 may need a `DeserializeFailed` variant if the codec needs to distinguish "valid JSON but not a JsonRpcMessage" from "invalid JSON"). If so, add:
```rust
#[error("valid JSON but not a JSON-RPC message: {0}")]
NotJsonRpc(String),
```

**7. Update test constructors**: Tests currently use `serde_json::json!(1)` for ids. Update to `Some(RequestId::Number(1))`. Tests using `serde_json::json!("ok")` for results stay as-is since result is still Value.

**8. Add round-trip tests** (per D-08): Add round-trip tests for RequestId (both variants), JsonRpcRequest (with and without params), JsonRpcResponse (success and error), JsonRpcMessage (both variants).
