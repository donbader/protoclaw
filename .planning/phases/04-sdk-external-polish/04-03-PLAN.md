    Enable `#![warn(missing_docs)]` on all 9 internal crates that don't have it yet (per D-03). The 4 SDK crates already have it.

    For each crate's lib.rs, add `#![warn(missing_docs)]` at the top (after any existing `#![...]` attributes).

    Then run `cargo doc --no-deps --workspace 2>&1 | grep "warning: missing"` to get the full list of undocumented items.

    Add doc comments to ALL public items that produce warnings (per D-04 — meaningful WHY comments, not "Returns the foo" stubs):

    **Per-crate guidance for doc comment style:**

    - `anyclaw` (binary crate): Module-level docs on lib.rs. Public re-exports get brief docs.
    - `anyclaw-core`: Manager trait methods explain the lifecycle contract (start → run → shutdown). Backoff/CrashTracker explain the failure recovery strategy. SessionStore explains persistence contract.
    - `anyclaw-config`: Config structs explain what each field controls and its default. Load function explains the layering order (defaults → YAML → env).
    - `anyclaw-jsonrpc`: Codec explains the framing protocol (line-delimited JSON). Types explain JSON-RPC 2.0 semantics.
    - `anyclaw-agents`: Manager explains ACP lifecycle. Connection explains subprocess management. Explain WHY bridge-collapsed architecture was chosen.
    - `anyclaw-channels`: Manager explains session-keyed routing. Connection explains port discovery. SessionQueue explains two-phase collect+flush.
    - `anyclaw-tools`: Manager explains MCP host lifecycle. External explains subprocess MCP servers. WasmRunner explains sandboxing strategy.
    - `anyclaw-supervisor`: Explain boot order invariant (tools → agents → channels) and WHY. ManagerSlot explains crash isolation.
    - `anyclaw-test-helpers`: Brief docs — test utility crate.

    For `pub(crate)` items that trigger warnings: add brief docs or use `#[allow(missing_docs)]` with a comment if the item is truly internal plumbing.

    IMPORTANT: Do NOT add `#[allow(missing_docs)]` broadly. Each allow must have a justification comment. The goal is zero warnings, not zero warnings via suppression.

    1. "No shared mutable state between managers" → `crates/anyclaw-core/src/manager.rs` at Manager trait definition
    2. "No anyhow in library crates" → Each library crate's error.rs or lib.rs
    3. "No bare .unwrap() in production code" → Already enforced by clippy — add comment at workspace lint config
    4. "No mod.rs files" → Root Cargo.toml or workspace lint config
    5. "No println! or log crate" → Workspace lint config or tracing setup
    6. "Do not change MANAGER_ORDER" → `crates/anyclaw-supervisor/src/lib.rs` at MANAGER_ORDER constant
    7. "Do not call run() without start()" → `crates/anyclaw-core/src/manager.rs` at Manager trait run() method
    8. "Do not call run() twice" → `crates/anyclaw-agents/src/manager.rs` (and channels/tools) at cmd_rx.take()
    9. "Do not access binary/env/working_dir on AgentConfig directly" → `crates/anyclaw-config/src/types.rs` at AgentConfig struct
    10. "No std::env::var in channel/tool binaries" → `crates/anyclaw-sdk-channel/src/lib.rs` or harness.rs
    11. "No cross-manager crate imports" → `crates/anyclaw-core/src/lib.rs` or manager.rs
    12. "ChannelEvent lives in anyclaw-sdk-types" → `crates/anyclaw-core/src/lib.rs` at re-export
    13. "ACP wire types live in anyclaw-sdk-types" → `crates/anyclaw-agents/src/lib.rs` at re-export
