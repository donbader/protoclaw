    **Step 2: Create `crates/anyclaw-sdk-types/src/proptest_impls.rs`.**

    This file contains:
    1. `prop_compose!` strategies for all public wire types
    2. Property tests using `proptest!` macro

    Wire types needing Arbitrary strategies (from codebase inspection):

    **ACP types (acp.rs):**
    - `ClientCapabilities` — experimental: Option<HashMap<String, Value>>
    - `InitializeParams` — protocol_version: u32, capabilities, options: Option<HashMap>
    - `InitializeResult` — protocol_version: u32, agent_capabilities: Option<AgentCapabilities> (use None for simplicity — AgentCapabilities is from external crate)
    - `McpServerInfo` — name, server_type, url, command, args, env, headers
    - `SessionNewParams` — session_id: Option<String>, cwd, mcp_servers
    - `SessionNewResult` — session_id
    - `ContentPart` — Text { text } | Image { url }
    - `SessionPromptParams` — session_id, prompt: Vec<ContentPart>
    - `SessionCancelParams` — session_id
    - `SessionForkParams` — session_id
    - `SessionForkResult` — session_id
    - `SessionListParams` — empty struct
    - `SessionListResult` — sessions: Vec<SessionInfo>
    - `SessionInfo` — session_id, metadata: HashMap<String, Value>
    - `SessionLoadParams` — session_id, cwd: Option, mcp_servers: Option
    - `ToolCallStatus` — enum with 4 variants (Pending, InProgress, Completed, Failed)

    Note: Skip `SessionUpdateType` and `SessionUpdateEvent` — the `#[serde(tag = "sessionUpdate")]` internally-tagged enum with `#[serde(flatten)]` on some variants (ConfigOptionUpdate, SessionInfoUpdate) makes round-trip testing unreliable due to serde flatten limitations. The existing hand-written round-trip tests in acp.rs already cover these thoroughly.

    **Channel types (channel.rs):**
    - `ChannelCapabilities` — streaming, rich_text: bool
    - `ChannelInitializeParams` — protocol_version, channel_id, ack: Option, options: HashMap
    - `ChannelInitializeResult` — protocol_version, capabilities
    - `DeliverMessage` — session_id, content: Value
    - `PeerInfo` — channel_name, peer_id, kind
    - `ChannelSendMessage` — peer_info, content
    - `ThoughtContent` — session_id, update_type, content
    - `AckNotification` — session_id, channel_name, peer_id, message_id: Option
    - `AckLifecycleNotification` — session_id, action
    - `ChannelAckConfig` — reaction, typing: bool, reaction_emoji, reaction_lifecycle
    - `ChannelRespondPermission` — request_id, option_id
    - `SessionCreated` — session_id, peer_info

    **Permission types (permission.rs):**
    - `PermissionOption` — option_id, label
    - `PermissionRequest` — request_id, description, options: Vec
    - `PermissionResponse` — request_id, option_id
    - `ChannelRequestPermission` — request_id, session_id, description, options: Vec

    **Session key (session_key.rs):**
    - `SessionKey` — via SessionKey::new(channel, kind, peer)

    **Strategy approach:**
    - Use `prop_compose!` for each struct
    - For String fields: use `"[a-zA-Z0-9_-]{1,20}"` regex strategy (avoids empty strings that might cause issues, keeps test output readable)
    - For Option<String>: use `proptest::option::of("[a-zA-Z0-9_-]{1,20}")`
    - For Vec<T>: use `proptest::collection::vec(arb_t(), 0..3)` (small vecs to keep tests fast)
    - For HashMap<String, Value>: use `proptest::collection::hash_map("[a-z]{1,5}", arb_json_value(), 0..3)`
    - For `serde_json::Value`: create a simple `arb_json_value()` strategy that generates: null, bool, i64, string, small arrays, small objects (no deeply nested structures — keep it bounded)
    - For `ToolCallStatus`: use `prop_oneof!` over the 4 variants
    - For `ContentPart`: use `prop_oneof!` over Text and Image
    - For `AgentCapabilities` (external crate): skip — use `Option::None` in InitializeResult strategy

    **Step 3: Add module declaration to lib.rs.**

    Add to `crates/anyclaw-sdk-types/src/lib.rs`:
    ```rust
    #[cfg(test)]
    mod proptest_impls;
    ```

    **Step 4: Property tests (per D-04).**

    For each wire type, write a `proptest!` test that:
    1. Generates an arbitrary instance
    2. Serializes to JSON string via `serde_json::to_string(&val).unwrap()`
    3. Deserializes back via `serde_json::from_str::<T>(&json).unwrap()`
    4. Asserts `original == restored`

    Group tests by module: `proptest! { #[test] fn acp_initialize_params_round_trips(val in arb_initialize_params()) { ... } }`

    Use 256 cases per test (proptest default) — sufficient for wire type testing without being slow.
