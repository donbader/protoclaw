pub fn resolve_all_binary_paths(config: &mut crate::AnyclawConfig) {
    let ext = config.extensions_dir.clone();
    for agent in config.agents_manager.agents.values_mut() {
        match &mut agent.workspace {
            crate::WorkspaceConfig::Local(local) => {
                local.binary.0[0] = resolve_binary_path(&local.binary.0[0], &ext);
            }
            crate::WorkspaceConfig::Docker(docker) => {
                // Docker entrypoint resolution...
            }
        }
    }
    for ch in config.channels_manager.channels.values_mut() {
        ch.binary = resolve_binary_path(&ch.binary, &ext);
    }
    for tool in config.tools_manager.tools.values_mut() {
        if let Some(ref mut bin) = tool.binary {
            *bin = resolve_binary_path(bin, &ext);
        }
    }
}

    ```rust
    /// Load per-extension sidecar defaults and merge into entity options.
    ///
    /// For each agent/channel/tool with a resolved binary path, checks for
    /// `<binary_path>.defaults.yaml`. If found, parses it as a flat YAML map
    /// and merges into the entity's `options` HashMap. User-provided options
    /// take precedence over extension defaults.
    ///
    /// Called after `resolve_all_binary_paths()` in `Supervisor::new()`.
    pub fn load_extension_defaults(config: &mut crate::AnyclawConfig) {
        // For each agent (Local workspace only — skip Docker per D-09):
        //   1. Extract binary path from local.binary.0[0]
        //   2. Build sidecar path: format!("{}.defaults.yaml", binary_path) (per D-01)
        //   3. If sidecar file doesn't exist → continue (per D-02)
        //   4. Read file contents, parse with serde_yaml::from_str::<HashMap<String, serde_json::Value>>()
        //   5. If parse fails → tracing::warn!("...") and continue (per discretion)
        //   6. For each (key, value) in parsed map:
        //      - Only insert if key NOT already in agent.options (per D-05, D-06 — user wins)
        //      - Use entry API: agent.options.entry(key).or_insert(value)
        //   7. Log at trace level when sidecar found and loaded (per discretion)
        //
        // Repeat same pattern for channels (ch.binary) and tools (tool.binary if Some)
    }
    ```

    The function iterates three entity types following the exact same pattern as `resolve_all_binary_paths()` in `resolve.rs`:
    - Agents: match on `WorkspaceConfig::Local` → use `local.binary.0[0]`, skip `Docker` (per D-09)
    - Channels: use `ch.binary` directly
    - Tools: use `tool.binary` if `Some`, skip if `None`

    Helper function `merge_sidecar_into_options(binary_path: &str, options: &mut HashMap<String, serde_json::Value>)` to avoid repeating the read/parse/merge logic three times. This helper:
    1. Constructs sidecar path: `format!("{binary_path}.defaults.yaml")`
    2. Checks `std::path::Path::new(&sidecar_path).exists()` — return early if false
    3. Reads file with `std::fs::read_to_string`
    4. Parses with `serde_yaml::from_str::<HashMap<String, serde_json::Value>>()`
    5. On parse error: `tracing::warn!(path = %sidecar_path, error = %e, "skipping malformed extension defaults")`
    6. On success: `tracing::trace!(path = %sidecar_path, keys = ?map.keys().collect::<Vec<_>>(), "loaded extension defaults")`
    7. Merges: `for (k, v) in map { options.entry(k).or_insert(v); }`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | extension_defaults.rs | mitigate | Sidecar files only merge into the `options` HashMap — structural fields (binary, enabled, args, workspace) are never modified. User config always overrides extension defaults via `entry().or_insert()`. |
| T-11-02 | Tampering | extension_defaults.rs | accept | A malicious actor with write access to the extension binary directory could place a crafted sidecar file. Accepted: if an attacker has write access to the binary directory, they can already replace the binary itself — the sidecar adds no new attack surface. |
| T-11-03 | Denial of Service | extension_defaults.rs | mitigate | Malformed sidecar YAML is caught by serde_yaml parse error, logged as warning, and skipped — does not crash the supervisor or block startup. |
| T-11-04 | Information Disclosure | extension_defaults.rs | accept | Sidecar path logged at trace level reveals binary location. Accepted: binary paths are already logged during resolution and visible in config. |
</threat_model>
