  <action>
**1. Install cargo-llvm-cov** (per D-08):
```bash
cargo install cargo-llvm-cov --locked
```
If already installed, this is a no-op.

**2. Run baseline coverage report:**
```bash
cargo llvm-cov --workspace --text 2>&1 | tee .planning/phases/01-tooling-lint-infrastructure/coverage-baseline.txt
```

This runs all unit tests across the workspace with source-based instrumentation and outputs a text summary showing per-file and total line coverage.

Note: Integration tests (`-p integration`) require pre-built binaries and may not work under llvm-cov instrumentation. Run workspace unit tests only (the default).

**3. Extract the TOTAL line coverage percentage** from the report. Look for the `TOTAL` row in the output.

**4. Document the coverage floor recommendation:**
- If total coverage is above 50%, recommend setting the floor at 5% below current (e.g., 62% measured → 57% floor)
- If total coverage is below 50%, recommend setting the floor at the current value (don't allow regression)
- Write the recommendation as a comment at the top of the baseline file:
```
# Baseline coverage captured: {date}
# Total line coverage: {X}%
# Recommended CI floor: {Y}% (current minus 5% buffer for refactoring headroom)
```

**5. Verify cargo-llvm-cov also supports HTML output** (useful for developers):
```bash
cargo llvm-cov --workspace --html --output-dir target/llvm-cov-html 2>/dev/null && echo "HTML report: target/llvm-cov-html/index.html" || echo "HTML output not available"
```
This is informational only — the HTML report is gitignored via target/.
  </action>
