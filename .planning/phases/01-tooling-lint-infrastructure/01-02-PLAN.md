<task type="auto">
  <name>Task 1: Fix all clippy warnings across workspace crates</name>
  <files>crates/**/*.rs</files>
  <read_first>
    - AGENTS.md §Conventions (unwrap rules: .expect("reason") for invariants, ? for fallible, bare .unwrap() only in tests)
    - AGENTS.md §Anti-Patterns (no unsafe, no mod.rs, no println in library code)
    - .planning/codebase/CONVENTIONS.md §Error Handling (thiserror in libraries, anyhow only at entry points)
  </read_first>
  <action>
**1. Run clippy and capture all warnings:**
```bash
cargo clippy --workspace 2>&1 | grep "warning\[" | sort | uniq -c | sort -rn
```

**2. Fix warnings by category, following these rules:**

- **`clippy::unwrap_used`**: In production code, replace `.unwrap()` with `.expect("reason")` explaining the invariant, or use `?` if the function returns Result. In test code (`#[cfg(test)]` modules), `.unwrap()` is allowed — add `#[allow(clippy::unwrap_used)]` on the test module if needed.

- **`clippy::needless_pass_by_value`**: Change function signatures to take `&T` instead of `T` where the value is only read. Be careful with trait implementations — don't change signatures required by traits.

- **`clippy::implicit_clone`**: Replace `.clone()` on references with `.to_owned()` or `.to_string()` as appropriate. Or restructure to avoid the clone.

- **`clippy::redundant_closure_for_method_calls`**: Replace `|x| x.method()` with `Type::method` or the appropriate method reference.

- **`clippy::manual_let_else`**: Convert `let x = match ... { Some(v) => v, None => return }` to `let Some(x) = ... else { return };`

- **`clippy::cloned_instead_of_copied`**: Replace `.cloned()` with `.copied()` on iterators of Copy types.

- **`clippy::flat_map_option`**: Replace `.flat_map(|x| x)` or `.filter_map(|x| x)` patterns with `.flatten()`.

- **`clippy::semicolon_if_nothing_returned`**: Add trailing semicolons to expression statements in blocks that return `()`.

- **`clippy::disallowed_types` (serde_json::Value)**: For EXISTING usages, add `#[allow(clippy::disallowed_types)]` with a comment like `// Grandfathered: typed replacement in Phase {N}` per D-03. Do NOT refactor these now — that's Phase 2-4 work.

- **`unused_imports`**: Remove the unused import. If it's a re-export used by downstream crates, verify with `cargo check --workspace` before removing.

- **`dead_code`**: Remove if truly unused. If it's a public API that may be used externally (SDK crates), keep it and add `#[allow(dead_code)]` with a comment.

**3. After each crate batch, verify:**
```bash
cargo clippy --workspace 2>&1 | grep "warning" | wc -l
```
Continue until zero warnings.

**4. Run `cargo fmt` after all changes** to ensure formatting stays clean:
```bash
cargo fmt --all
```

**Important constraints:**
- Do NOT change any public API signatures in SDK crates (sdk-types, sdk-agent, sdk-channel, sdk-tool) — these are published crates
- Do NOT remove `pub` items from SDK crates even if they appear unused locally — external consumers may use them
- Do NOT change `MANAGER_ORDER` or boot/shutdown sequences
- Preserve all existing test behavior — run `cargo test --workspace` after changes
  </action>
